# Release Checklist

## GitHub

- Remove or ignore local logs, pid files, and runtime state
- Copy `config.example.json` to `config.json` locally and set real paths
- Confirm README matches current setup flow
- Add screenshots to the repository
- Create a tag for the release version

## ClawHub

- Prepare cover image and screenshots
- Prepare short and long listing descriptions
- Verify install flow from a clean machine
- Verify default config works without local private paths
- Attach release notes and support link
