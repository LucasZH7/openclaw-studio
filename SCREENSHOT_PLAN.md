# Screenshot Plan

## Goal

Prepare a small but strong set of visuals that makes the product legible in under 10 seconds on GitHub and ClawHub.

## Required Shots

1. Hero dashboard
- Show avatar, title, status, and score
- Theme should be clean and balanced
- Use a friendly but serious robot style

2. Trend and metrics
- Show the efficiency chart with hover tooltip
- Keep score and token lines both enabled
- Include one or two metrics cards in frame

3. Chat and control
- Show the chat panel plus nudge / heal controls
- Make it obvious that this is interactive, not just decorative

4. Customization
- Open settings drawer
- Show avatar controls, headwear, hands, wings, and background effects

5. Personality shot
- Focus on the avatar rail
- Pick one charming customization state
- This is the “cute robot” marketing shot

## Visual Rules

- Avoid overly busy particle backgrounds in screenshots
- Use a balanced theme with readable contrast
- Prefer states that look “alive” but not chaotic
- Keep layout clean; do not show broken experiments or edge-case poses

## File Naming

- `01-hero-dashboard.png`
- `02-trend-and-metrics.png`
- `03-chat-and-controls.png`
- `04-customization-drawer.png`
- `05-avatar-personality.png`

  avatar idle motion -> nudge button -> score / status feedback
